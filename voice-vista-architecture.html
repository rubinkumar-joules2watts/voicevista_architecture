Audio ingestion — two ingest paths converging to S3
Telephony System
Hooks into call centre telephony (SIP / PSTN). Streams live audio in real time to the ingestion layer for immediate STT processing.
Audio Upload API
REST endpoint accepting .wav / .mp3 files for async batch processing. Validates format, duration, and file size before queuing to Celery.
Call Audio Files (S3)
Raw audio persisted to AWS S3 with lifecycle policies. Retrieved by the STT engine. Source of truth for call replay and compliance audits.

AI & Analytics — three-stage intelligence pipeline
Speech-to-Text (STT)
Converts raw call audio to time-stamped transcripts. Speaker diarization separates agent vs customer turns. Supports multiple accents and languages via Azure Speech SDK.
Text Analytics (NLP)
Runs NLP pipeline on transcripts: sentiment scoring per sentence, named entity recognition, topic classification, and intent extraction for downstream engines.
Azure OpenAI (GPT-4o)
LLM backbone for generative tasks — quality audit scoring, escalation risk explanation, revenue opportunity identification, and personalised training recommendations.

Processing engines — four parallel AI-powered analysis modules + configurable persona signals
Quality Audit Engine
Auto-generates quality scorecards per call. Checks adherence to scripts, compliance clauses, and SLA thresholds. Score stored in RDS for QA Analyst review and override.
Escalation & Risk Detection
Detects high-risk sentiment spikes, regulatory trigger phrases, and legal red flags in real time. Fires alerts immediately to Supervisors via the Notification layer.
Upsell & Revenue Engine
Identifies call moments with upsell opportunity signals — positive sentiment, product mentions, buying intent. Surfaces ranked revenue insights on the dashboard.
Training Recommendation Engine
Analyses agent performance trends across calls. Identifies skill gaps and generates personalised coaching paths surfaced to Supervisors on the dashboard.
⚙ Persona Configurations — pluggable signal detectors that feed every engine above
😐 Tonality Detector
Classifies the overall vocal tone of each speaker turn — Aggressive, Calm, Assertive, Defensive, or Neutral. Drives quality scoring weights and escalation sensitivity thresholds.
🎭 Emotion Recognition
Multi-label emotion tagging per utterance — Frustrated, Happy, Anxious, Confused, Satisfied. Layered on top of STT output using fine-tuned transformer models.
📊 Sentiment Scoring
Sentence-level and turn-level sentiment (Positive / Negative / Neutral) with polarity scores. Tracks sentiment arc across the call to detect satisfaction trends and inflection points.
😏 Sarcasm Detection
Identifies sarcastic or passive-aggressive phrasing using contextual LLM inference. Flags calls where surface-level sentiment contradicts the emotional subtext for QA review.
🗣️ Urgency Signal
Rates customer urgency (Critical / High / Low) based on speech rate, keyword density, and emotional intensity. Directly feeds the Escalation engine's routing decision.
🤝 Empathy Level
Scores agent empathy expression per interaction — acknowledgement phrases, active listening cues, and validating language. Key metric for Training Recommendation and quality audit.
💡 Config pattern: Each persona signal is a independently toggleable detector. Supervisors & QA Analysts configure which signals are active per campaign or call type — the selected set is injected as a context persona into the GPT-4o prompt, shaping how each engine interprets and weights its output.

Data Storage — three-tier persistence strategy
AWS S3
Object store for raw audio files and full-length transcripts. Lifecycle policies auto-archive cold data. Versioned for compliance retention requirements.
AWS DynamoDB
Low-latency NoSQL for real-time event streams — alert states, live call metadata, agent status. Single-digit ms reads for dashboard real-time feeds.
AWS RDS (PostgreSQL)
Relational store for structured audit records, QA scores, call metadata, agent profiles, and aggregated analytics consumed by the Dashboard layer.

Notifications — multi-channel alert delivery
Escalation Notifications
Triggered by the Escalation & Risk Detection engine. Sent to Supervisors immediately via email, in-app toast, and mobile push. Includes call ID, transcript snippet, and risk reason.
AWS Communication Services
AWS SNS for fan-out pub/sub, SES for transactional email, and SMS gateway for urgent text alerts. Configurable per actor role and severity level.

Dashboard & Reporting — six insight views surfaced to actors
Call Insights
Per-call breakdown: duration, sentiment arc, topics discussed, agent talk ratio, silence periods, and key moments.
Audit Scores
AI-generated quality scorecard with pass/fail per criterion. QA Analysts can override scores and add review comments.
Centralized Dashboard
Executive-level aggregated view across teams and date ranges — call volumes, avg scores, escalation rates, and SLA trends.
Escalations
Live feed and historical log of escalation events. Linked to call recordings and agent profiles for direct action.
Training Gaps
Aggregated skill gap heatmap by agent and team. Drill-down to specific calls that demonstrate the identified weakness.
Revenue Opportunities
List of calls flagged with upsell signals, ranked by propensity score. Exportable for sales team follow-up actions.

Security monitoring — cross-cutting concerns across all layers
Application Insights
APM tracing for all FastAPI services. Tracks request latency, error rates, and dependency calls. Feeds CloudWatch for unified observability across the platform.
AWS Secrets Manager
Centralised vault for API keys, database credentials, and third-party tokens. Automatic rotation. Zero hard-coded secrets in code or environment files.
AWS CloudWatch
Aggregates logs and metrics from EKS pods, RDS, and DynamoDB. Alarm triggers for anomaly detection, pod crash loops, and budget overruns.
Role-Based Access Control
IAM roles enforce least-privilege per actor. Agents can't see each other's data; QA Analysts have read-only call access; Leadership sees aggregate views only.
